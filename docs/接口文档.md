# ChuHanAI 阶段一 MVP 服务端接口文档（联调版）

本文档仅覆盖阶段一 P0 能力：匿名会话、匹配、对局、计时、悔棋/求和/认输、聊天、断线重连。

## 1. 协议与版本

- 实时主通道：`WS /ws/v1/game`
- 健康检查：`GET /api/v1/health`
- 协议版本：`X-Protocol-Version: 1.0`
- 客户端版本：`X-Client-Version: <semver>`
- 兼容策略：
  - 向后兼容新增字段：仅追加，不改名不改语义
  - 破坏性变更：升级大版本路径（如 `/ws/v2/game`）

## 2. 鉴权与请求头

阶段一无账号体系，采用匿名会话鉴权。

### 2.1 首次连接（无会话）

握手请求头：

- `X-Protocol-Version`（必填）
- `X-Client-Version`（必填）
- `X-Trace-Id`（选填，建议 UUID）

握手成功后服务端返回：

- `X-Session-Id`
- `X-Resume-Token`

### 2.2 重连（恢复会话）

握手请求头：

- `X-Session-Id`（必填）
- `X-Resume-Token`（必填）
- `X-Protocol-Version`（必填）
- `X-Client-Version`（必填）
- `X-Trace-Id`（选填）

鉴权失败返回 HTTP `401` 并关闭连接。

## 3. 通用消息封装

所有 WS 消息统一 JSON 封装：

```json
{
  "msgId": "m-20260314-000001",
  "seq": 101,
  "ackSeq": 100,
  "type": "MOVE_REQUEST",
  "roomId": "R202603140001",
  "sessionId": "S_ANON_001",
  "timestampMs": 1773475200123,
  "payload": {}
}
```

字段约定：

- `msgId`：客户端全局唯一（建议 ULID/UUID），用于幂等
- `seq`：客户端单调递增序列
- `ackSeq`：已收到的服务端连续序号确认
- `timestampMs`：Unix Epoch 毫秒（UTC）
- `roomId`：无房间时可为 `null`

## 4. 响应包装与错误模型

### 4.1 统一响应包装（服务端回包/广播）

```json
{
  "msgId": "srv-20260314-001",
  "replyToMsgId": "m-20260314-000001",
  "type": "MOVE_ACCEPTED",
  "roomId": "R202603140001",
  "sessionId": "SERVER",
  "timestampMs": 1773475200456,
  "success": true,
  "error": null,
  "payload": {}
}
```

错误时：

```json
{
  "msgId": "srv-20260314-002",
  "replyToMsgId": "m-20260314-000001",
  "type": "MOVE_REJECTED",
  "roomId": "R202603140001",
  "sessionId": "SERVER",
  "timestampMs": 1773475200460,
  "success": false,
  "error": {
    "code": "MOVE_ILLEGAL",
    "message": "炮走法不合法",
    "retryable": false
  },
  "payload": {
    "authoritativeFen": "..."
  }
}
```

### 4.2 HTTP 状态码（握手与健康检查）

- `200`：成功
- `400`：请求格式错误
- `401`：会话鉴权失败
- `426`：协议版本不支持（需升级客户端）
- `429`：限流
- `500`：系统异常

## 5. MVP 接口清单（WS）

### 5.1 匹配

#### A1. 加入匹配队列

- 用途：进入 FIFO 匹配
- 客户端消息：`MATCH_JOIN`
- 服务端回包：`MATCH_JOINED`（入队成功）或错误
- 服务端异步事件：`MATCH_SUCCESS`（匹配成功）

`payload` 请求：

```json
{
  "clientVersion": "0.1.0"
}
```

`MATCH_SUCCESS.payload`：

```json
{
  "roomId": "R202603140001",
  "mySide": "RED",
  "opponentSessionId": "S_ANON_002",
  "baseTimeMs": 600000,
  "incrementMs": 15000,
  "initialFen": "..."
}
```

#### A2. 取消匹配

- 用途：退出匹配队列
- 客户端消息：`MATCH_CANCEL`
- 服务端回包：`MATCH_CANCELLED`

### 5.2 对局走子

#### B1. 走子请求

- 用途：提交一步走子，由服务端权威裁决
- 客户端消息：`MOVE_REQUEST`
- 服务端回包：`MOVE_ACCEPTED` 或 `MOVE_REJECTED`
- 服务端广播：`TIME_SYNC`（可与 `MOVE_ACCEPTED` 同包）

`payload` 请求：

```json
{
  "moveNo": 12,
  "from": "b2",
  "to": "b9",
  "piece": "CANNON"
}
```

`MOVE_ACCEPTED.payload`：

```json
{
  "moveNo": 12,
  "from": "b2",
  "to": "b9",
  "piece": "CANNON",
  "capturedPiece": null,
  "boardFenAfter": "...",
  "nextTurn": "BLACK",
  "redTimeLeftMs": 598000,
  "blackTimeLeftMs": 600000
}
```

`MOVE_REJECTED.payload`：

```json
{
  "authoritativeFen": "...",
  "expectedMoveNo": 12
}
```

### 5.3 对局控制

#### C1. 悔棋申请

- 请求：`UNDO_REQUEST`
- 对手收到事件：`UNDO_PENDING`
- 响应：`UNDO_RESPONSE`
- 结果事件：`UNDO_APPLIED` 或 `UNDO_REJECTED`

`UNDO_RESPONSE.payload`：

```json
{
  "decision": "ACCEPT"
}
```

#### C2. 求和申请

- 请求：`DRAW_REQUEST`
- 对手收到事件：`DRAW_PENDING`
- 响应：`DRAW_RESPONSE`
- 结果：同意则 `GAME_OVER(endReason=DRAW_AGREED)`，拒绝则 `DRAW_REJECTED`

#### C3. 认输

- 请求：`RESIGN`
- 结果：立即 `GAME_OVER(endReason=RESIGN)`

### 5.4 聊天

#### D1. 发送聊天

- 请求：`CHAT_SEND`
- 回包：`CHAT_ACCEPTED` 或错误
- 广播：`CHAT_BROADCAST`

`payload` 请求：

```json
{
  "content": "你好，开始吧"
}
```

`CHAT_BROADCAST.payload`：

```json
{
  "messageId": "c-001",
  "senderSessionId": "S_ANON_001",
  "content": "你好，开始吧",
  "sentAtMs": 1773475201888
}
```

### 5.5 重连与同步

#### E1. 快照同步

- 用途：断线重连后恢复权威状态
- 请求：`SNAPSHOT_SYNC`
- 回包：`SNAPSHOT_SYNCED`

`payload` 请求：

```json
{
  "lastKnownMoveNo": 25
}
```

`SNAPSHOT_SYNCED.payload`：

```json
{
  "room": {
    "roomId": "R202603140001",
    "status": "PLAYING",
    "currentTurn": "RED",
    "boardFen": "...",
    "moveNo": 26,
    "redTimeLeftMs": 410000,
    "blackTimeLeftMs": 389000
  },
  "recentMoves": [
    {
      "moveNo": 26,
      "from": "e3",
      "to": "e4",
      "piece": "PAWN",
      "createdAtMs": 1773475201555
    }
  ],
  "pendingControlEvent": null
}
```

#### E2. 心跳

- 客户端：`PING`
- 服务端：`PONG`
- 心跳建议间隔：`5s`
- 读空闲超时：`15s`（超时断链）

### 5.6 对局结束

#### F1. 对局结束广播

- 事件：`GAME_OVER`
- 触发：`CHECKMATE` / `RESIGN` / `TIMEOUT` / `DRAW_AGREED` / `DRAW_STALEMATE`

`payload`：

```json
{
  "winnerSide": "RED",
  "endReason": "CHECKMATE",
  "finalMoveNo": 88,
  "finalFen": "...",
  "endedAtMs": 1773475299999
}
```

## 6. 错误码定义（MVP）

通用：

- `INVALID_REQUEST`：字段缺失/格式错误
- `UNAUTHORIZED`：`sessionId` 或 `resumeToken` 无效
- `UNSUPPORTED_VERSION`：协议版本不兼容
- `RATE_LIMITED`：触发限流
- `SYSTEM_ERROR`：系统异常
- `REQUEST_TIMEOUT`：请求处理超时

匹配：

- `MATCH_ALREADY_IN_QUEUE`：重复入队
- `MATCH_NOT_IN_QUEUE`：取消时不在队列

对局：

- `ROOM_NOT_FOUND`：房间不存在
- `ROOM_FINISHED`：房间已结束
- `TURN_CONFLICT`：非当前回合
- `MOVE_ILLEGAL`：走子非法
- `MOVE_NO_CONFLICT`：`moveNo` 与服务端不一致
- `IDEMPOTENT_REPLAY`：重复消息（返回历史结果）

控制：

- `CONTROL_CONFLICT`：已有未决控制请求
- `CONTROL_TIMEOUT`：对手未在时限内响应

聊天：

- `CHAT_TOO_LONG`：消息超长
- `CHAT_RATE_LIMITED`：发送过快

重连：

- `RECONNECT_WINDOW_EXPIRED`：超出重连窗口
- `SNAPSHOT_OUTDATED`：客户端状态过旧，需全量同步

## 7. 分页、时间、枚举约定

- 分页：MVP 主链路无分页接口；`SNAPSHOT_SYNCED.recentMoves` 固定最多返回最近 `20` 步
- 时间：统一 `timestampMs`（Epoch 毫秒，UTC）
- 坐标：`a0` 到 `i9`
- 阵营：`RED` / `BLACK`
- 房间状态：`WAITING` / `PLAYING` / `FINISHED` / `CANCELLED`

## 8. 幂等、限流、重试、超时约束

### 8.1 幂等

- 幂等键：`(sessionId, msgId)` + 业务域隔离（`MATCH/ROOM/CHAT`）
- 幂等适用：`MATCH_JOIN`、`MOVE_REQUEST`、`UNDO_REQUEST`、`DRAW_REQUEST`、`RESIGN`、`CHAT_SEND`
- 重复请求：返回首次处理结果，不重复执行状态变更

### 8.2 限流

- 聊天：单连接 `5s` 内最多 `5` 条
- 控制消息：单房间单方 `2s` 内最多 `3` 条
- 超限统一返回：`RATE_LIMITED` 或 `CHAT_RATE_LIMITED`
- 聊天长度上限：`300` 字符

### 8.3 重试

- 客户端在 `retryable=true` 时可重试
- 重试必须复用同一 `msgId`（否则视为新请求）
- 指数退避建议：`200ms -> 500ms -> 1s`，最多 3 次

### 8.4 超时

- 请求处理超时：默认 `3s`，返回 `REQUEST_TIMEOUT`
- 控制请求等待对手响应：默认 `15s`，超时按拒绝处理
- 重连窗口：固定 `30s`（超时判负）

## 9. 最小 HTTP 接口（运维/联调辅助）

### H1. 健康检查

- `GET /api/v1/health`
- 200 响应示例：

```json
{
  "status": "UP",
  "serverTimeMs": 1773475200000,
  "version": "0.1.0"
}
```

## 10. 文档冲突处理结果（已定稿）

1. 聊天长度上限采用 `300`（与方案库表一致）。
2. 终局原因包含 `DRAW_STALEMATE`（客户端统一展示“和棋”）。
3. 重连窗口固定 `30s`，超时判负。
