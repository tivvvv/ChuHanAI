# ChuHanAI 使用说明

本文档面向当前阶段一 MVP 版本，目标是帮助你快速完成本地启动、联调和基础功能验证。

## 1. 当前可用范围

当前代码已经打通以下最小可玩闭环：

- 服务端启动与健康检查
- 客户端匿名会话建立
- WebSocket 实时连接
- 双端匹配进入对局
- 基础走子与服务端权威裁决
- 计时同步
- 认输结束对局
- 对局结束后再次匹配

当前仍建议你重点按本文档做一次人工验证：

- 两个客户端窗口是否都能正常进入大厅
- 是否能成功匹配到同一房间
- 是否能正常走子、认输、回到大厅再来一局

## 2. 环境要求

- JDK 21
- Maven 3.9+
- MySQL 8.x

## 3. 数据库准备

服务端默认连接以下数据库：

- 数据库地址：`jdbc:mysql://localhost:3306/chu_han_ai`
- 用户名：`root`
- 密码：`root`

你可以先执行：

```sql
CREATE DATABASE IF NOT EXISTS chu_han_ai
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_general_ci;
```

服务端启动时会自动执行 `schema.sql` 初始化表结构。

如果你不想使用默认配置，可以通过环境变量覆盖：

- `DB_URL`
- `DB_USERNAME`
- `DB_PASSWORD`

## 4. 服务端启动

在项目根目录执行：

```bash
mvn -pl chuhanai-server spring-boot:run -DskipTests
```

默认端口：

- HTTP：`8080`
- 实时 WebSocket：`10090`

启动成功后可访问健康检查：

```bash
curl http://localhost:8080/api/v1/health
```

预期返回类似：

```json
{"status":"UP","serverTimeMs":1773500000000,"version":"0.1.0"}
```

## 5. 客户端启动

在项目根目录执行：

```bash
mvn -pl chuhanai-client-javafx javafx:run
```

如果你要同时启动两个客户端做对战测试，建议开两个终端，分别执行同一个命令：

```bash
mvn -pl chuhanai-client-javafx javafx:run
```

```bash
mvn -pl chuhanai-client-javafx javafx:run
```

## 6. 客户端连接配置

客户端默认配置如下：

- HTTP 基地址：`http://localhost:8080`
- 健康检查：`http://localhost:8080/api/v1/health`
- WebSocket 地址：`ws://localhost:10090/ws/v1/game`
- 协议版本：`1.0`

可用以下环境变量或 JVM 参数覆盖：

- `CHUHANAI_HTTP_BASE_URL`
- `CHUHANAI_HEALTH_URL`
- `CHUHANAI_WS_URL`
- `CHUHANAI_REALTIME_PORT`
- `CHUHANAI_PROTOCOL_VERSION`
- `CHUHANAI_CLIENT_VERSION`

例如：

```bash
CHUHANAI_HTTP_BASE_URL=http://localhost:38080 \
CHUHANAI_REALTIME_PORT=39090 \
mvn -pl chuhanai-client-javafx javafx:run
```

## 7. 基础使用流程

### 7.1 启动后

客户端启动后会先进行：

1. 健康检查
2. WebSocket 实时连接
3. 匿名会话建立
4. 自动进入大厅

如果本地保存过上一次的匿名会话，客户端会尝试恢复；若服务端没有对应房间，会自动回到大厅。

### 7.2 开始一局

1. 在两个客户端中都进入大厅
2. 分别点击“开始匹配”
3. 匹配成功后进入棋盘页面
4. 由红方先走
5. 双方轮流落子

### 7.3 结束一局

当前最稳定的结束方式是：

1. 任意一方点击“认输”
2. 双方收到对局结束结果
3. 点击“返回大厅”或“再来一局”

### 7.4 再来一局

点击“再来一局”后，客户端会重新发起匹配请求。当前版本已经修复了“旧房间状态未清理导致无法再次匹配”的问题。

## 8. 建议验证清单

建议你按下面顺序人工验证：

1. 服务端能否正常启动
2. `GET /api/v1/health` 是否返回 `UP`
3. 两个客户端是否都能进入大厅
4. 两个客户端是否能匹配成功
5. 红方第一步 `a3 -> a4` 是否能正常走出
6. 双方是否都能看到棋盘变化
7. 一方认输后双方是否都收到结束结果
8. 点击“再来一局”后是否还能再次匹配

## 10. 常见问题

### 10.1 客户端连不上服务端

优先检查：

- 服务端是否真的已经启动
- `8080` 和 `10090` 端口是否被其他旧进程占用
- 客户端是否连接到了错误端口

可手动检查：

```bash
curl http://localhost:8080/api/v1/health
```

如果健康检查正常，但客户端仍连接失败，通常是旧服务还在占用实时端口，导致客户端连到过期实现。

### 10.2 WebSocket 握手异常

如果出现类似“不是合法 WebSocket 响应”的问题，通常说明：

- `10090` 上跑的不是当前最新版本的服务端
- 或者仍有旧的测试/临时服务进程未停止

建议先停止旧进程，再重新启动当前代码。

### 10.3 数据库连接失败

请确认：

- MySQL 已启动
- `chu_han_ai` 数据库已创建
- `DB_URL`、`DB_USERNAME`、`DB_PASSWORD` 配置正确

### 10.4 启动时报端口被占用

你可以改用其他端口启动服务端：

```bash
mvn -pl chuhanai-server spring-boot:run -DskipTests -Dspring-boot.run.arguments="--server.port=38080 --netty.port=39090"
```

然后用客户端环境变量指向新端口：

```bash
CHUHANAI_HTTP_BASE_URL=http://localhost:38080 \
CHUHANAI_REALTIME_PORT=39090 \
mvn -pl chuhanai-client-javafx javafx:run
```
